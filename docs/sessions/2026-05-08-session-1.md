## Что сделано
- Структура проекта по ТЗ, инициализирован git (12 коммитов).
- Зависимости: `flask`, `sqlalchemy`, `python-dotenv`, `httpx`, `pillow` — без расширений сверх ТЗ.
- БД SQLite + 3 модели (`Post`, `Rubric`, `GenerationLog`), автосид 4 рубрик при первом запуске.
- `ClaudeClient` через ProxyAPI (`/anthropic/v1/messages`, Bearer + `x-api-key`).
- `KlingImageProvider`: JWT HS256 формируется руками через `hmac+base64` (без `pyjwt`), `создание задачи → поллинг → скачивание` в `static/uploads/<uuid>.png`.
- `ManualUploadHandler`: валидация формата/размера, ресайз до 2000px через PIL.
- 4 рубрики со стартовыми системными промптами и шаблонами image-промптов в `core/prompts.py`.
- Flask-API: `/api/rubrics`, `/api/generate-text`, `/api/generate-image`, `/api/upload-image`, CRUD `/api/posts`, `GET/PATCH /api/settings/rubrics/...`.
- Frontend: 1 HTML, 3 вкладки на ванильном JS, превью VK, библиотека с поиском и фильтром, редактор промптов рубрик.
- Брендовый CSS с дизайн-токенами (кремовый/горчичный/тёмно-зелёный), шрифты Lora + Inter.
- Логирование AI-вызовов в `logs/ai_calls.jsonl`.
- README с пошаговой инструкцией для непрограммиста.

## Ключевые решения
1. **JWT без `pyjwt`.** HS256 на 15 строках через `hmac` + `base64` — не пришлось расширять `requirements.txt`.
2. **ProxyAPI авторизация дублируется:** `Authorization: Bearer ...` + `x-api-key: ...` — обе формы поддерживаются ProxyAPI, страхуем от изменений в их прокси-логике.
3. **Дефолтный регион Kling — Singapore (`api-singapore.klingai.com`).** Российские пользователи регистрируются на `app.klingai.com/global/dev` → их ключи живут в международном регионе. Сделан настраиваемым через `KLING_BASE_URL`.
4. **Стартовая рубрика не выбрана** — пользователь должен явно ткнуть в одну из 4. Защита от случайной генерации «не той» рубрики.
5. **Ресайз ручной загрузки до 2000px** по большей стороне — экономит место без потери качества для VK.
6. **Ошибки наружу — понятные русские строки, без stack trace.** Любая ошибка идёт в JSONL с `success=false`.

## Возникшие проблемы и как решились
| Проблема | Что показал Kling | Решение |
|---|---|---|
| Документация ProxyAPI не показала готовый пример Anthropic-эндпоинта | — | Использован `/anthropic/v1/messages` + комбинированный заголовок Bearer/x-api-key. Подтверждено успешной генерацией текста. |
| Документация Kling не открылась через WebFetch (HTTP 446) | — | Использованы community-материалы; flow стандартный. |
| Первый вызов Kling: `code 1002, access key not found` | 401 | Дефолтный endpoint был `api.klingai.com` (китайский), а ключи пользователя — с международного дашборда. Переключили на `api-singapore.klingai.com` + сделали настраиваемым через `.env`. Коммит `5bc3d80`. |
| Второй вызов Kling: `code 1102, Account balance not enough` | 429 | Не баг — на аккаунте Kling не было оплаченного пакета Image Generation. После пополнения баланса генерация прошла. |

## Идеи и заметки на v0.2
- **Публикация в VK** через VK API (long-lived community token) — отдельная вкладка «Опубликовать».
- **Расписание** публикаций: APScheduler + Windows Scheduled Task.
- **Контент-план на месяц**: календарная сетка с drag-n-drop черновиков.
- **Альтернативный image-провайдер** (Recraft / Ideogram / Stability) — `ImageProvider` уже абстрактный, добавить ~30 строк. Полезно как fallback при сбоях/исчерпании баланса Kling.
- **Учёт стоимости** в `cost_estimate_rub` — по актуальным тарифам ProxyAPI и Kling.
- **Несколько вариантов текста** на один черновик с выбором лучшего.
- **Точная ширина превью** ~510px как в реальном VK для пиксель-перфектной проверки.
