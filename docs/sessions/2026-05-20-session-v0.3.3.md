## Хронология

1. **Восстановление сессии** после прерывания v0.3.2. Проверено состояние: 12 файлов изменены, не закоммичены; backend smoke прошёл; тег ждёт.
2. **Шкала оценок -2..+2 вместо 1..5.** Реализация + миграция данных (4 строки, MAX>2). Фикс визуального бага: ререндер контрола после клика.
3. **Проблема генерации картинок.** Падало с 502 / timeout. Discovery подтвердил что `gpt-image-2` — правильная модель (вышла 2026-04-21, OpenAI флагман, ProxyAPI поддерживает). Причина: `REQUEST_TIMEOUT_SEC=180` мало для `quality=high`. Поднят до 360.
4. **Референс Лоры — Discovery.** Через subagent с web search. Подтверждено:
   - OpenAI `/v1/images/edits` принимает до 16 референсов (multipart `image[]`), `/generations` — нет.
   - Claude Vision: image-блок в messages, рекомендация Anthropic «картинка ПЕРЕД текстом».
   - Kling 3.0 Web UI: фича «Subject Library» + «Element Binding» — Dr. Nik делает руками.
5. **Получение ассетов.** Figma MCP rate-limit. Dr. Nik выгрузил 7 PNG (4096×4096, ~10 МБ) в `core/assets/` — эмоции Лоры.
6. **Реализация Lora reference pipeline** в одной фазе:
   - `scripts/optimize_lora_assets.py` — 4K PNG → 1024 JPG (~150 КБ).
   - `core/lora_references.py` — реестр 7 эмоций + маппинг 8 рубрик → эмоция.
   - `core/llm_client.py` — поддержка `image_b64` в `ClaudeClient.generate()`.
   - `core/prompt_generator.py` — принимает `rubric_key`/`emotion`/`use_reference`, отправляет картинку Claude.
   - `core/image_clients.py` — новый метод `generate_with_reference()` использует `/v1/images/edits`.
   - `routes/api.py` — все три эндпоинта (prompt, image, list-emotions) прокидывают параметры.
   - `templates/index.html` + `static/js/app.js` — дропдаун «Эмоция Лоры» во вкладках Видео/Изображение.
7. **Smoke-тесты:**
   - Claude увидел Лору и описал детали с картинки (зелёные глаза, золотые серьги, цветочный воротник) — не из общего шаблона.
   - `/edits` с `quality=low` прошёл за <60с, с `quality=high` и длинным промптом тоже OK.
   - Один транзитный 500 от ProxyAPI «Internal Server Error received from provider» — при повторе всё прошло.
   - Визуально: первые сгенерированные изображения с референсом — внешний вид Лоры стабилен (рыжие кудри, горчичный свитер).
8. **Баг «залипания» кнопки.** `setBusy(btn, true, ...)` сохранял `_origHTML` каждый вызов, перезаписывая исходную метку при двойном вызове («Готовлю промпт…» → «Генерирую изображение…»). После `setBusy(btn, false)` восстанавливалась НЕ оригинальная метка а «Готовлю промпт…». Пофикшено: сохраняем `_origHTML` только если ещё не сохранён.

## Ключевые решения

1. **Маппинг рубрик → эмоция** статичный в коде, не настраиваемый из UI. Принцип: business_english/grammar→explains, common_mistake→corrects, mini_fact→surprise, word_of_day/slang→praises, остальное→greetings.
2. **Дропдаун эмоции** показывается во вкладках Видео и Изображение (там нет рубрики поста). На «Создать пост» референс не подцеплен — медиа из публикации убраны после v0.3.0 pivot.
3. **Опция «— без референса —»** есть для случая когда нужна нейтральная картинка без Лоры.
4. **Оптимизация ассетов** — отдельный скрипт, не делается на лету. Идемпотентен по mtime. Запускается руками при добавлении новых PNG.
5. **Шкала оценок -2..+2 vs 1..5.** Семантическая разница: «-2 в корзину» гораздо информативнее чем «1 звезда из 5». Для будущего «auto-modification промпта по фидбеку» (Вариант C) это даст чёткий сигнал «не учиться на этом».
6. **Транзитный 500 от ProxyAPI** — не добавлен ретрай. Принцип: не строим инфраструктуру под гипотетический сбой. Если повторится систематически — добавим.
7. **Баг setBusy** — исправлен сохранением `_origHTML` только при первом вызове.
