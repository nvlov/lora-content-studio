## Что сделано

### Блок A — UX-полировка
- Иконочные кнопки (inline SVG из Lucide-style набора, без сторонних зависимостей)
- Универсальная модалка подтверждения `confirmModal()` → Promise<boolean>, Esc/Enter
- Soft-delete: добавлено `Post.deleted_at`, удалённые не показываются, эндпоинт `POST /api/posts/<id>/restore`
- Прогресс-индикаторы: спиннер + блок Kling с таймером «Идёт N сек… Это занимает 1–2 минуты, не закрывай вкладку»
- Счётчик символов VK (4096) с цветовой индикацией: нейтральный → горчица 3500+ → ошибка 4000+ + блокировка кнопок публикации
- Toast-система: контейнер в правом нижнем углу, очередь, типы success/warn/error/info
- Hot-keys: `Ctrl+Enter` (генерировать), `Ctrl+S` (сохранить), `Ctrl+P` (опубликовать), `Esc` (закрыть модалку)
- `beforeunload` при несохранённых изменениях
- Фокус-стили (`:focus-visible`) и `prefers-reduced-motion`

### Блок B — Расширение рубрик до 8
- Добавлены: `business_english` 💼, `slang_informal` 🎭, `mini_fact` 🤔, `free_topic` ✨ (системные промпты по ТЗ)
- `seed_new_rubrics_v0_2()` в `core/db.py` — идемпотентный, не перезаписывает уже существующие
- В UI рубрика `free_topic` подменяет узкое поле «Тема» на большую `textarea` «Описание задачи»; backend передаёт это описание как `user_message` для Claude

### Блок C — Медиа-студия
- Новые модели: `MediaAsset` (image/video, source, размер, размеры, mime), `MediaPrompt` (idea_ru, prompt_en, style, aspect, best_for)
- `core/prompt_generator.py` — Claude возвращает строго JSON-массив из 3 промптов с фирменными ограничениями (палитра, no readable text)
- Эндпоинты: `POST/GET/DELETE /api/media`, `POST /api/media/generate-prompts`, `GET/POST/DELETE /api/media/prompts`
- UI «Медиа»: drag-n-drop зона + сетка медиа-карточек + блок генератора промптов с радио image/video, дропдаунами стиля и aspect-ratio, 3 карточки результата с кнопками «Копировать» и «Сохранить»
- Видео в редакторе: блок «Медиа поста» с тремя источниками (Kling / загрузка / выбор из библиотеки через модалку); превью VK поддерживает `<video controls>`

### Блок D — Публикация в VK
- `core/vk_client.py` — `VKClient` с методами `upload_photo`, `upload_video`, `post_to_wall`; русифицированные подсказки по типичным кодам ошибок (5, 7, 9, 14, 15, 100, 214)
- `core/scheduler.py` — APScheduler `BackgroundScheduler` + `SQLAlchemyJobStore` на той же `lora.db` (таблица `apscheduler_jobs`); восстановление задач при старте + догон пропущенных
- Эндпоинты: `POST /api/posts/<id>/publish-now`, `POST /api/posts/<id>/schedule`, `DELETE /api/posts/<id>/schedule`, `GET /api/posts/<id>/publication-history`, `GET /api/vk/status`, `POST /api/posts/<id>/duplicate`, `POST /api/posts/<id>/restore`
- UI: кнопки «Опубликовать сейчас» (с подтверждением и автооткрытием поста в новой вкладке) и «Запланировать» (нативный `datetime-local` picker, минимум +10 минут)
- В библиотеке цветные плашки статусов (`draft / scheduled / published / publish_failed`) + фильтр по статусу + кнопки «Открыть в ВК» (🔗), «Повторить публикацию» (для failed), «Отменить расписание» (для scheduled)

### Техдолг (фоном)
- Ручной HMAC-JWT для Kling заменён на `pyjwt`
- Kling timeout повышен до 180 сек, шаг поллинга — 5 сек (по требованию C3)
- Файлы изображений теперь сохраняются в `static/uploads/images/`, видео — в `static/uploads/videos/`
- Миграция `migrate_uploads_v0_2()` в `core/db.py` переносит файлы из старого расположения и обновляет `Post.image_path`

## Ключевые решения
- **APScheduler в одном процессе с Flask.** Job store на том же SQLite — это самое простое для single-user локального инструмента. Запуск только в основном процессе (проверка `WERKZEUG_RUN_MAIN`).
- **Pico-минимализм во фронте.** Никаких сборщиков, никаких фреймворков. SVG-иконки inline в JS-объекте `ICONS`, чтобы не подключать иконочные библиотеки и не плодить файлы.
- **Видео — не более одного на пост, не вместе с картинкой.** Это упрощает превью, миграцию и логику публикации в VK. `media_kind` авто-выставляется на бэкенде.
- **Soft-delete вместо физического удаления** — лучше потерянное место, чем потерянные данные.
- **Ручной ALTER TABLE без Alembic** — оставлено на v0.4 (как в плане), сейчас идемпотентная миграция через `PRAGMA table_info`.

## Возникшие проблемы и как решились
- **Артефакт-файлы pip.** При первом неэкранированном `pip install ...>=...` PowerShell интерпретировал `>=` как редирект и создал файлы `=2.8.0` и `=3.10.0` в корне. Удалены отдельным коммитом.
- **VK API нюансы.** Подтверждены параметры через документацию: `wall.post` для публикации от имени сообщества требует `owner_id = -group_id` и `from_group=1`. Видео в VK обрабатывается асинхронно после загрузки — `wall.post` всё равно сработает, VK покажет процесс обработки прямо в посте.
- **Время.** Все datetime в БД — UTC naive. Из API принимаем ISO 8601 (с/без `Z`), приводим к UTC. В UI отображаем через `toLocaleString('ru-RU')`.
