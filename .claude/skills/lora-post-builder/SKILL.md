# Lora Post Builder

## Контекст

Проект lora-content-studio (локальный путь: `C:\Users\nlovt\claude_projects\lora-content-studio\`) — Flask + SQLite инструмент для маркетинговой публикации в VK-сообщество «Лора | Английский с AI-репетитором» (продвижение продукта LinguaAI).

Бренд-персонаж — Лора. Тёплый, поддерживающий AI-репетитор английского для русскоязычных учеников уровней A1–C1. Тон: репетитор-друг. Без снобизма, без академичности, обращение на «ты». Полный бренд-голос — в `references/lora-voice-guide.md` проекта.

## Когда применять

- «Сделай пост Лоры про X»
- «Оформи этот факт/идиому/правило в пост»
- «Вот ссылка на статью — преврати в пост для рубрики Y»
- «Нужен пост на тему Z, рубрика свободная»
- Любой запрос на создание контента для VK-сообщества Лоры

## Когда НЕ применять

- Если пользователь просит опубликовать существующий пост — это делает `manage.py publish <id>` или Flask UI, не этот Skill.
- Если просит редактировать драфт в БД — открой Flask UI на localhost:5000.
- Для генерации видео в Kling Web — Skill только готовит `kling_prompt`, видео делается руками во внешнем сервисе.

## Вход

Один из:
1. **Текст идеи** в чате — Dr. Nik описывает тему.
2. **URL** — Dr. Nik даёт ссылку. Используй WebFetch для извлечения содержимого. Если WebFetch не дал результата или сайт за пейволом — попроси текст руками или запусти Agent с задачей поиска информации.
3. **Файл/фрагмент** в чате.

Если фактов не хватает — НЕ додумывай. В спорных местах ставь строку `[уточнить]`. Не выдумывай цифры, даты, цитаты, имена авторов, ссылки.

## Алгоритм

### Шаг 1. Определить рубрику

8 рубрик в проекте:

| key | name | формат |
|---|---|---|
| `word_of_day` | Слово дня | одно английское слово + перевод + 2 примера |
| `common_mistake` | Типичная ошибка | ❌ неправильно vs ✅ правильно + объяснение |
| `phrase_from_life` | Фраза из жизни | идиома или разговорное выражение + диалоги |
| `grammar_simple` | Грамматика на пальцах | правило простыми словами + 3 примера |
| `business_english` | Деловой английский | 3 рабочих шаблона + что не говорить |
| `slang_informal` | Сленг и неформальное | модное словечко + где уместно/нет |
| `mini_fact` | Минифакт | один лингвистический/культурный факт |
| `free_topic` | Свободная тема | произвольный пост по описанию пользователя |

Если рубрика не задана — выбери подходящую по содержанию входа или спроси Dr. Nik одной фразой («рубрика — `mini_fact` или `phrase_from_life`?»).

**Если нужен полный системный промпт рубрики** (точный формат поста, лимит символов, требования): прочитай `core/generators/prompts.py` в проекте — там константы `WORD_OF_DAY_PROMPT`, `COMMON_MISTAKE_PROMPT` и т. д. Каждая описывает:
- Точную структуру поста (нумерованные пункты)
- Лимит символов (важно для VK: 4096 максимум, по рубрикам 600–1000)
- Тон и требования
- Эмодзи-якорь в начале

**Не дублируй промпты сюда — читай актуальные из проекта.**

### Шаг 2. Подобрать эмоцию Лоры

7 эмоций для image-референса: `greetings`, `praises`, `corrects`, `explains`, `thinks`, `surprise`, `neutral`.

Маппинг рубрика → эмоция (по умолчанию):

| рубрика | эмоция |
|---|---|
| `business_english` | `explains` |
| `grammar_simple` | `explains` |
| `common_mistake` | `corrects` |
| `mini_fact` | `surprise` |
| `slang_informal` | `praises` |
| `word_of_day` | `praises` |
| `phrase_from_life` | `greetings` |
| `free_topic` | `greetings` |

Источник: `core/generators/lora_references.py` (`RUBRIC_TO_EMOTION` dict). Если хочешь сверить актуальный маппинг — прочитай файл.

Можно переопределить эмоцию явно если содержимое требует другой (например, серьёзный факт — `thinks` вместо дефолтной `surprise`).

### Шаг 3. Сгенерировать VK-текст

Следуй ТОЧНОЙ структуре рубрики из `core/generators/prompts.py`. Соблюдай:
- Лимит символов (указан в каждом промпте)
- Эмодзи-якорь в начале (`🇬🇧` для word_of_day, `💡` для common_mistake и т. д. — см. поле `emoji` в STARTER_RUBRICS / NEW_RUBRICS_V0_2)
- Обращение на «ты»
- Тёплый репетиторский тон
- Финальный призыв/вопрос для вовлечения

**Не используй markdown-форматирование (`**жирный**`, `_курсив_`).** VK его не рендерит, показывает звёздочки сырьём. Только plain-текст и эмодзи.

### Шаг 4. Опционально — Telegram-версия

Если Dr. Nik попросил («сделай и для Telegram») или если канал настроен (`TELEGRAM_BOT_TOKEN` в `.env` проекта):
- Длина 700–1500 символов
- Структура близка к VK, но первые 1-2 предложения важнее — Telegram не обрезает, но превью в списке чатов берёт первые символы
- Без хэштегов внутри текста — Telegram-канал бренда LinguaAI не использует хэштеги в стиле
- Можно чуть более разговорно чем VK

Если флага не дано — `telegram` секцию в JSON не включай.

### Шаг 5. Сгенерировать image_prompt

Для генерации картинки через gpt-image-2 (`/v1/images/edits` с референсом эмоции Лоры).

**Базовый шаблон по рубрике** — см. константы `IMG_*` в `core/generators/prompts.py` (например, `IMG_WORD_OF_DAY`, `IMG_COMMON_MISTAKE`). Подставь конкретную тему в `{topic}`.

Стиль бренда: soft Pixar-3D, cream + mustard color palette, dark green accents. Лора — рыжеволосый персонаж в горчичном свитере. Без читаемого текста на картинке (кроме явно указанного слова).

Если ничего из шаблонов не подходит — собери свой prompt по тому же стилю.

### Шаг 6. Опционально — kling_prompt

Для видео-генерации во внешнем Kling Web UI (генерация Kling делается руками, не из проекта).

Активируй если Dr. Nik попросил или если рубрика — `phrase_from_life` / `slang_informal` (контент хорошо смотрится как короткое видео). Структура промпта — см. `core/generators/prompt_generator.py` и `references/lora-voice-guide.md`.

### Шаг 7. Собрать JSON и сохранить

Файл: `data/inbox/text/YYYY-MM-DD-<rubric>-<slug>.json` (slug — kebab-case от темы, до 30 символов, транслит если кириллица).

Используй абсолютный путь от корня проекта: `C:\Users\nlovt\claude_projects\lora-content-studio\data\inbox\text\<file>`.

### Шаг 8. Отчёт пользователю

Сообщи в чате:
- Файл создан: путь
- Превью VK-текста (первые 200 символов)
- Рубрика и эмоция
- Команда импорта: `venv\Scripts\python manage.py import-from-json <path>` (по умолчанию импорт + генерация картинки)
- Если нужен только текст без картинки: `... --no-image`

## JSON Schema

Полная формальная схема: `docs/schemas/lora-post-v1.json` в проекте. Краткое представление:

```json
{
  "schema_version": "lora-post/v1",
  "source": {
    "type": "url|text|idea",
    "url": "https://... (если type=url)",
    "title": "Название источника или короткий заголовок",
    "fetched_at": "2026-05-26T16:00:00Z"
  },
  "rubric_key": "word_of_day|common_mistake|phrase_from_life|grammar_simple|business_english|slang_informal|mini_fact|free_topic",
  "topic": "Краткое описание темы (для UI и Kling-prompt)",
  "emotion": "greetings|praises|corrects|explains|thinks|surprise|neutral",
  "platforms": {
    "vk": {
      "content": "Полный текст VK-поста, plain-text без markdown"
    },
    "telegram": {
      "content": "Полный текст Telegram-поста (если делали)"
    }
  },
  "image_prompt": "Английский prompt для gpt-image-2, готовый к /v1/images/edits с референсом эмоции",
  "kling_prompt": "Английский prompt для Kling t2v (если рубрика требует видео)",
  "notes": "Опциональные комментарии для Dr. Nik (что было неясно, что [уточнить])"
}
```

Поля `telegram`, `kling_prompt`, `notes` — опциональные. Остальные — обязательные.

## Что НЕ делать

- **Не вызывай Anthropic API напрямую из Skill.** Текст генерируешь ты (Claude в текущей сессии) через собственное рассуждение.
- **Не вызывай ProxyAPI для генерации картинки.** Это делает `manage.py import-from-json` через `core/generators/image_clients.py` (там уже настроен `/v1/images/edits` с референсом эмоции).
- **Не публикуй пост.** Skill создаёт только драфт в JSON. Публикация — отдельный шаг через Flask UI или `manage.py publish`.
- **Не пиши в БД напрямую.** Только JSON-файл в `data/inbox/text/`. CLI команда сама импортирует.
- **Не выдумывай факты.** Если не уверен в этимологии, цитате, дате — ставь `[уточнить]` и упомяни в `notes`.
- **Не используй markdown-форматирование в тексте поста.** VK его не рендерит.
- **Не дублируй инструкции рубрик в SKILL.md** — читай актуальные из `core/generators/prompts.py`.
