# CLAUDE_CODE_VK_OAUTH_002 — Диагностика: VK_OAUTH_APP_ID не читается из .env

> **Тип:** микро-фикс к Phase 2 (диагностика конфигурации, не новая функциональность)
> **Целевая версия после:** v0.3.0-rc1 (без изменения версии)
> **Приоритет:** блокирующий — без фикса OAuth-скрипт не запускается

---

## 1. Симптом

При запуске `python scripts/get_vk_token.py` (после коммита Phase 2) скрипт **дважды** падает с ошибкой:

```
ОШИБКА: VK_OAUTH_APP_ID не задан в .env.
Заполни VK_OAUTH_APP_ID (App ID Standalone-приложения с id.vk.com)
и запусти снова.
```

Пользователь подтвердил что **строка `VK_OAUTH_APP_ID=54587420` в `.env` присутствует** (добавлена вручную в конец файла, сохранена через Ctrl+S в VS Code).

## 2. Что НЕ делать в ходе диагностики

- Не переписывать архитектуру `scripts/get_vk_token.py`
- Не добавлять новые переменные окружения
- Не править остальной OAuth-код (`core/vk_oauth.py`, `core/vk_client.py`)
- Не предполагать. Каждый шаг — проверка факта.

## 3. План диагностики

### Шаг 0 — Проверить факт наличия строки в файле

В терминале (PowerShell):

```powershell
Select-String -Path .env -Pattern "VK_OAUTH_APP_ID"
```

**Что покажет:**
- Если строка найдена — увидим её содержимое и номер строки
- Если ничего не выведено — строки в файле нет (значит файл сохранён не туда / редактируется не тот `.env`)
- Если найдено два совпадения — есть дубликат (это тоже проблема, последнее значение перетирает первое или наоборот в зависимости от логики dotenv)

Зафиксировать результат в отчёте отдельной строкой.

### Шаг 1 — Проверить кодировку и BOM

VS Code на Windows может сохранять файл с BOM или в UTF-16 — `python-dotenv` это не всегда корректно парсит.

```powershell
Get-Content .env -Encoding Byte -TotalCount 4 | ForEach-Object { '{0:X2}' -f $_ }
```

**Что искать:**
- `EF BB BF` в первых трёх байтах — UTF-8 BOM (может ломать dotenv в некоторых версиях)
- `FF FE` или `FE FF` — UTF-16 (точно ломает dotenv)
- Если первые байты — печатные ASCII (например, `23 20 50 52` для `# PR`) — кодировка чистая

### Шаг 2 — Проверить как именно `scripts/get_vk_token.py` загружает .env

В файле должно быть что-то вроде:
```python
from dotenv import load_dotenv
load_dotenv()
```

Но возможные подводные камни:
1. `load_dotenv()` без аргументов ищет `.env` в текущей рабочей директории — а это не обязательно директория скрипта
2. Пользователь запускает из `lora-content-studio/`, а скрипт лежит в `scripts/` — относительный путь может разъезжаться
3. Если скрипт делает `os.environ.get('VK_OAUTH_APP_ID')` ДО `load_dotenv()` — переменной не будет

Открыть `scripts/get_vk_token.py`, посмотреть:
- Где вызывается `load_dotenv()`
- Какие у неё аргументы
- В каком порядке относительно `os.environ.get()` или `os.getenv()`

### Шаг 3 — Прямая проверка через Python REPL

В терминале (там где запускается скрипт):

```powershell
python -c "from dotenv import load_dotenv; import os; load_dotenv(); print('VK_OAUTH_APP_ID =', repr(os.environ.get('VK_OAUTH_APP_ID')))"
```

**Что покажет:**
- `VK_OAUTH_APP_ID = '54587420'` — dotenv видит, проблема в логике скрипта (Шаг 4)
- `VK_OAUTH_APP_ID = None` — dotenv не видит, проблема в файле или пути (вернуться к Шагам 0–1)
- `VK_OAUTH_APP_ID = ''` — строка есть но значение пустое (опечатка в .env: пробелы вокруг `=`, кавычки)

### Шаг 4 — Найти конкретное место падения в скрипте

Если Шаг 3 показал что dotenv видит значение, но скрипт всё равно падает — значит в скрипте есть условие, которое неправильно определяет «не задан». Возможные причины:
1. Проверка строгая по типу: `if app_id == ""` вместо `if not app_id`
2. Скрипт вызывает свой собственный `config.py`, который мог не успеть прочитать `.env`
3. `load_dotenv()` вызывается с `override=False` (по умолчанию) и значение уже занято старым из системного env

Найти в `scripts/get_vk_token.py` строку которая выводит «ОШИБКА: VK_OAUTH_APP_ID не задан в .env» — посмотреть условие выше.

## 4. Фикс (минимальный)

В зависимости от того, что показала диагностика — одна из конкретных правок:

### Если Шаг 0 показал что строки нет в файле
- Пользователь сохранил `.env` в неправильное место (например, в `scripts/.env` вместо корня)
- Команда: показать пользователю абсолютный путь правильного `.env` и попросить вписать переменную ещё раз

### Если Шаг 1 показал BOM или UTF-16
- Переcохранить `.env` в UTF-8 без BOM через VS Code: правый-нижний угол → клик на «UTF-8 with BOM» → «Save with Encoding» → «UTF-8»

### Если Шаг 2/4 показал что `load_dotenv()` вызывается с неправильным путём
- Поправить в `scripts/get_vk_token.py`:
  ```python
  from pathlib import Path
  from dotenv import load_dotenv
  load_dotenv(Path(__file__).resolve().parent.parent / '.env')
  ```
- Это явно указывает на `.env` в корне проекта независимо от того, откуда запущен скрипт

### Если Шаг 3 показал что значение есть, но строгая проверка не пропускает
- Поправить условие в `scripts/get_vk_token.py` на `if not app_id:` (truthy-check вместо строгого сравнения)
