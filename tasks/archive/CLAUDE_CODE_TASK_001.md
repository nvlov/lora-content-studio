**Пользователь:** Dr. Nik — не разработчик, продуктовый специалист. Все действия и команды должны сопровождаться короткими понятными комментариями на русском. Код тоже комментируется на русском в ключевых местах.

**Назначение продукта:** одностраничный локальный веб-инструмент для генерации маркетинговых постов сообщества ВКонтакте `vk.com/lora_ai_english` (продвижение продукта **LinguaAI** — AI-репетитора английского языка по имени **Лора**, лендинг `lora-aitutor.ru`).

**Целевая аудитория контента:** широкая — все, кто интересуется изучением и разговорной практикой английского языка (уровни A1–C1).

**Что именно делает инструмент в этой итерации (v0.1):**
1. Принимает на входе выбор рубрики и (опционально) тему.
2. Генерирует текст поста через Claude Sonnet 4.5 (через ProxyAPI).
3. Генерирует иллюстрацию через Kling AI API ИЛИ позволяет загрузить готовое изображение вручную.
4. Показывает превью «как будет выглядеть в ВК».
5. Даёт отредактировать текст и регенерировать/заменить картинку.
6. Сохраняет пост как черновик в локальной БД SQLite.
7. Отображает библиотеку сохранённых черновиков с поиском.

**Что НЕ делается в v0.1** (важно — не отвлекаться):
- Публикация в ВК (это v0.2)
- Расписание публикаций (v0.2)

- Контент-план на месяц (v0.2)

### Шаг 3. Конфиг и БД

   **Post:**
   - `id` (PK, int)

### Шаг 5. Image-клиенты (KlingAI + Manual Upload)

1. **Перед написанием кода:** через web search изучи актуальные документы Kling AI API (`https://docs.qingque.cn/...` или официальную документацию Kling) и подтверди:
   - endpoint для text-to-image генерации
   - схему аутентификации (Kling использует JWT-токен, подписанный на основе access_key и secret_key — реализуй формирование JWT строго по их документации)
   - доступные модели и параметры (соотношение сторон, стиль)
   - структуру ответа (URL результата, поллинг статуса задачи)
2. В `core/image_clients.py` создать абстрактный базовый класс `ImageProvider` с методом:
   ```python
   def generate(self, prompt: str, aspect_ratio: str = "1:1") -> str:
       """Возвращает локальный путь к сохранённой картинке (в static/uploads/)"""
   ```
3. Реализовать `KlingImageProvider(ImageProvider)`:
   - Формирует JWT
   - Создаёт задачу text-to-image
   - Поллит статус задачи (с разумным timeout 60 сек, опрос каждые 3 сек)
   - Скачивает готовое изображение в `static/uploads/{uuid}.png`
   - Возвращает относительный путь
4. Реализовать `ManualUploadHandler` (не наследник, отдельный класс):
   - Принимает загруженный файл от Flask request
   - Валидирует тип (jpg/png/webp) и размер (макс 10 MB)
   - Сохраняет в `static/uploads/{uuid}.{ext}` через PIL (опционально ресайз до 2000px по большей стороне)
   - Возвращает путь
5. Каждая операция логируется в `ai_calls.jsonl`.
6. Коммит: `feat: image providers — kling and manual upload`.

### Шаг 7. Flask-роуты (backend)

Создать `routes/api.py` с эндпоинтами:

| Метод | URL | Назначение | Тело запроса | Ответ |
|---|---|---|---|---|
| `GET` | `/` | главная страница | — | HTML |
| `GET` | `/api/rubrics` | список рубрик | — | `[{key, name, emoji}, ...]` |
