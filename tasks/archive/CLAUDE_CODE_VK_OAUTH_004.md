### Шаг 3 — Проверить актуальные scope через VK API

Создать **одноразовый** скрипт `scripts/check_vk_scope.py` (≤30 строк), запустить, **удалить после теста**:

```python
import os
from dotenv import load_dotenv
import httpx

load_dotenv()
token = os.environ['VK_USER_ACCESS_TOKEN']
v = os.environ.get('VK_API_VERSION', '5.131')
group_id = int(os.environ['VK_GROUP_ID'])

# 1. Какие scope реально выданы (bitmask)
r1 = httpx.get(
    'https://api.vk.ru/method/account.getAppPermissions',
    params={'access_token': token, 'v': v},
    timeout=15
).json()
print("account.getAppPermissions:", r1)

# 2. Реальный access к фото-upload (без upload, только URL)
r2 = httpx.get(
    'https://api.vk.ru/method/photos.getWallUploadServer',
    params={'access_token': token, 'group_id': group_id, 'v': v},
    timeout=15
).json()
print("photos.getWallUploadServer:", r2)

# 3. Реальный access к video.save (без upload, только заявка)
r3 = httpx.get(
    'https://api.vk.ru/method/video.save',
    params={'access_token': token, 'group_id': group_id, 'name': 'test', 'v': v},
    timeout=15
).json()
print("video.save:", r3)
```

**Интерпретация:**
- `account.getAppPermissions` возвращает bitmask. Биты: photos=4, video=16, wall=8192, groups=262144, offline=65536. **Сумма этих битов**, если все scope активны: `4 + 16 + 8192 + 65536 = 73748` (без groups; с groups — `335884`).
- `photos.getWallUploadServer` → возвращает `response` с `upload_url` = ✅ работает, scope `photos` активен
- `photos.getWallUploadServer` → error code 15 (subcode 1133) = ❌ scope `photos` не выдан
- `video.save` → возвращает `response` с `upload_url` = ✅ работает, scope `video` активен
- `video.save` → error code 1051 = ❌ video.save заблокирован на уровне типа приложения (известное ограничение Standalone-приложений через id.vk.com)

После запуска **зафиксировать результат в отчёте**, удалить скрипт.
