## Сессия 1: v0.1 MVP — локальная генерация постов
**Дата:** 2026-05-08
**Длительность:** ~1 час (автогенерация Claude Code)
**Версия после сессии:** v0.1.0

### Что сделано
- Создана полная структура проекта согласно ТЗ `tasks/CLAUDE_CODE_TASK_001.md`.
- Инициализирован git, 11 осмысленных коммитов по шагам ТЗ.
- Настроены зависимости (`flask`, `sqlalchemy`, `python-dotenv`, `httpx`, `pillow`) и шаблон `.env.example` с инструкцией.
- БД SQLite с тремя моделями: `Post`, `Rubric`, `GenerationLog`. При первом запуске `init_db()` засеивает 4 стартовые рубрики.
- Клиент `ClaudeClient` для ProxyAPI (`/anthropic/v1/messages`, Bearer + `x-api-key`, `anthropic-version: 2023-06-01`). Все ошибки (нет ключа, сетевые, нерабочий ответ) дают понятный текст пользователю и пишутся в `ai_calls.jsonl`.
- Клиент `KlingImageProvider`: формирование JWT (HS256, claims `iss`/`exp`/`nbf`) вручную через `hmac+base64` (без сторонней библиотеки), создание задачи на `/v1/images/generations`, поллинг статуса (3 сек, до 120 сек), скачивание результата в `static/uploads/<uuid>.png`.
- Класс `ManualUploadHandler` для ручной загрузки с валидацией формата/размера и ресайзом до 2000px по большей стороне через PIL.
- Системные промпты 4 рубрик и шаблоны промптов для иллюстраций — в `core/prompts.py`.
- Flask-роуты: `GET /`, `GET /api/rubrics`, `POST /api/generate-text`, `POST /api/generate-image`, `POST /api/upload-image`, CRUD `/api/posts`, `GET/PATCH /api/settings/rubrics/...`.
- Frontend: одна HTML-страница, 3 вкладки, ванильный JS с состоянием в одном объекте `state`, тосты для ошибок, превью «как в ВК» с белой карточкой и квадратной картинкой.
- Брендовый CSS с дизайн-токенами (кремовый/горчичный/тёмно-зелёный), pill-кнопки, шрифты Lora + Inter через Google Fonts.
- README с пошаговой инструкцией для непрограммиста.

### Ключевые решения
- **JWT для Kling — без `pyjwt`.** Стандартный HS256 формируется в ~15 строк через `hmac` и `base64`. Это позволяет не расширять `requirements.txt` сверх списка из ТЗ и не тащить зависимость ради одного места.
- **ProxyAPI авторизация:** в заголовки кладутся одновременно `Authorization: Bearer ...` и `x-api-key: ...`. ProxyAPI принимает обе формы — это страхует от изменений в их прокси-логике.
- **Kling text2image flow:** create-задача → поллинг GET `/v1/images/generations/{task_id}` → скачивание картинки локально. Картинки кладутся под uuid, чтобы не было коллизий и можно было кэшировать.
- **Стартовая рубрика не выбрана.** Пользователь обязан явно ткнуть в одну из 4 — это уменьшает шанс случайно сгенерировать «не ту» рубрику.
- **Ресайз загружаемых картинок до 2000px** по большей стороне — экономит место и при этом сохраняет качество для ВК.
- **Ошибки наружу — понятные русские строки.** Никаких stack trace в UI. Любая ошибка от Claude/Kling логируется в `ai_calls.jsonl` с success=false.

### Возникшие проблемы и как решились
- Документация ProxyAPI на момент написания не показывала явный пример запроса к Anthropic. Решил по логике (`/anthropic/v1/messages`) и комбинированному заголовку — готов оперативно поправить, если ProxyAPI вернёт ошибку endpoint.
- Официальная документация Kling не открылась через WebFetch (возвращала HTTP 446). Использовал community-материалы и подтверждённые в нескольких источниках факты: JWT с claims `iss/exp/nbf`, HS256, эндпоинт `/v1/images/generations`, model_name=`kling-v1`.

### Что не удалось / отложено
- Сквозной end-to-end тест с реальными ключами (у Claude Code их нет). Пользователь должен прогнать смоук-тесты локально — см. список в финальном отчёте.
