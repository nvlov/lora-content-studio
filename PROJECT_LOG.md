## Текущая версия
**v0.4.0** — структурный рефакторинг под расширение SMM:
- `core/` декомпозирован на `publishers/` (vk + telegram заглушка + youtube/tiktok placeholders), `generators/` (Claude + gpt-image-2), `storage/` (SQLAlchemy), `ingest/` + `analytics/` (пустые пакеты под перспективу).
- CLI `manage.py` — параллельный к Flask UI терминальный интерфейс.
- Мультиплатформенная публикация: `Post.target_platforms` (JSON) + таблица `post_publications`.
- Бренд-ассеты вынесены из кодовой папки: `core/assets/` → `assets/lora/`.
- Корень проекта подчищен: новые папки `references/`, `briefs/`, `tmp/`; `tasks/` разбит на `active/` и `archive/`; `docs/` сгруппирован по типу.

## Предыдущая версия
**v0.3.3** — две независимые части:
- **Шкала оценок** переведена с 1..5 на −2..+2 (−2 неприемлемо / 0 нейтрально / +2 в канал). Семантика «в корзину» вместо «1 звезда». Миграция данных через `MAX(rating)>2` идемпотентна.
- **Референс Лоры в pipeline.** 7 эмоций (приветствие/похвала/поправка/объяснение/размышление/удивление/нейтральная), маппинг на 8 рубрик. Claude получает image-блок и описывает Лору с картинки; gpt-image-2 переключён на `/v1/images/edits` с приложенным референсом — внешний вид персонажа стабилизируется между генерациями. Workflow: пользователь выбирает эмоцию из дропдауна во вкладках Видео/Изображение → Claude пишет промпт «увидев» эту эмоцию → gpt-image-2 рендерит с тем же файлом как референсом. Оригиналы 4K используются для ручной загрузки в Kling Subject Library.

В промежуточном v0.3.1 (без тега) сделаны: UI strip медиа из формы поста, markdown-фильтр для VK-постов (`**жирный**` → `жирный`), удалён legacy `/api/upload-image`.

## Как добавить новую сессию
1. Скопировать [`docs/sessions/_template.md`](docs/sessions/_template.md) в файл `docs/sessions/YYYY-MM-DD-session-N.md`.
2. Заполнить разделы (цель, что сделано, решения, проблемы, идеи).
3. Добавить строку в таблицу выше и в `docs/sessions/README.md`.
4. Закоммитить и запушить.
